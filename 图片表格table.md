**表3-2 大语言模型结构化抽取 Prompt 核心规则及说明**

| 规则要求 | 设计目的 |
|---|---|
| 仅提取本期值、期末值，不混入同比或上期数 | 防止横向对比数据干扰当前入库结果 |
| 无法确认时返回 `null`，不得猜测或补造 | 抑制大模型幻觉，保证数据可信 |
| 所有金额统一换算为“万元” | 统一量纲，便于后续查询与计算 |
| 去除千分位逗号、百分号等格式字符 | 便于数值清洗和 Decimal 规范化 |
| 严格按照预定义 JSON 字段输出，不新增字段 | 防止结构漂移和幻觉字段入库 |
| 不允许输出股票代码、简称等身份字段 | 避免模型误填身份信息，由主表统一回填 |

**表4-1 财报问询主要查询类别及必需槽位定义表**

| 查询类型 | 触发关键词示例 | 必需识别槽位 | 应用场景 |

**表4-2 数据图表智能推荐策略与判别规则**

| 图表类型 | 触发判别规则 | 展示优势 |
|---|---|---|
| 折线图 | 趋势查询且时间序列数据点 >= 3 | 直观展示指标随时间的变化趋势 |
| 单柱状图 | 单指标排名查询，类别数量较少 | 强调不同主体间的数值高低 |
| 横向柱状图 | 排名查询且主体名称较长 | 提升标签可读性，适合 TopN 排名 |
| 分组柱状图 | 多公司同指标对比，或单公司多指标对比 | 适合并列比较多个对象 |
| 堆叠柱状图 | 组成结构型指标对比 | 同时展现总量与内部构成 |
| 雷达图 | 多指标 + 多公司矩阵型比较 | 适合展示综合能力轮廓差异 |
| 饼图 | 单期结构占比分析，类别数较少 | 突出各部分占总体的贡献比例 |
| 面积图 | 连续时间序列且强调累计变化 | 兼顾趋势与规模感表达 |
| 双折线图 | 两家公司或两个指标的同期趋势对比 | 适合观察同步变化和分化拐点 |

**表5-1 多意图执行计划标准步骤定义表**

**表5-2 四层递进式智能校验拦截体系**

| 校验层级 | 校验标的与参数 | 检测方式 | 目的 |
|---|---|---|---|
| 完整性校验 | 检查执行计划是否存在缺失步骤、依赖项是否齐全 | 遍历 DAG 步骤状态与 depends_on 关系 | 防止遗漏关键执行环节 |
| 一致性校验 | 检查跨步骤输出的同主键数据是否冲突 | 主键比对、结果集交叉校验 | 防止前后数值口径不一致 |
| 合理性校验 | 检查同比增速、利润率等是否超出合理阈值，如 \|同比\| > 1000% | 阈值规则、区间判断、异常值识别 | 防止极端异常值直接进入答案 |
| 引用有效性校验 | 若输出含“主要因为、导致、受益于”等归因词，则必须绑定 Reference | 归因关键词正则匹配 + 引用对象非空判断 | 防止无依据归因和模型幻觉 |
